# ОптСнаб — демо интернет-магазина оптовой торговли

Демо-сайт оптового магазина канцелярии и сантехники. Сделан как статический сайт — **работает на GitHub Pages без сервера и без базы данных**. Каталог и цены обновляются загрузкой Excel-файла прямо в браузере.

## Что внутри

- **Витрина** (`index.html`) — каталог с категориями, поиском, сортировкой, карточкой товара и корзиной.
- **Оптовая заявка** — корзина с учётом минимальных партий и форма запроса (без онлайн-оплаты).
- **Демо-админка** (`admin.html`) — загрузка прайса в `.xlsx`, предпросмотр и обновление каталога.

## Как обновлять цены (главное)

1. Открыть страницу **«⚙️ Прайс»** (`admin.html`).
2. Нажать **«Скачать шаблон прайса»** — получится Excel со всеми колонками.
3. Заполнить своими товарами и ценами, сохранить.
4. Перетащить файл в окно загрузки → проверить предпросмотр → **«Применить»**.
5. Каталог в магазине обновляется сразу. Программист не нужен.

### Формат Excel

Первая строка — заголовки. Порядок колонок неважен, важны названия:

| Колонка | Обяз. | Описание |
|---|---|---|
| `Артикул` | да | Уникальный код товара |
| `Наименование` | да | Название |
| `Категория` | да | Канцелярия / Сантехника / Бумага / Хозтовары и т.д. |
| `Цена` | да | Оптовая цена, число |
| `Единица` | нет | шт., упак., пачка… |
| `Мин.партия` | нет | Минимальный заказ, число |
| `Остаток` | нет | Кол-во на складе |
| `Описание` | нет | Текст описания |

Готовый пример — файл [`Прайс_пример.xlsx`](Прайс_пример.xlsx) в репозитории.

## Публикация на GitHub Pages

1. Создать репозиторий и загрузить все файлы (сохранив структуру папок).
2. **Settings → Pages → Source: `Deploy from a branch` → ветка `main`, папка `/ (root)`.**
3. Через минуту сайт будет доступен по адресу `https://<ваш-логин>.github.io/<репозиторий>/`.

Файл `.nojekyll` уже добавлен, чтобы GitHub Pages отдавал папку `assets` без обработки.

## Важно про демо-режим

Это **демонстрация**, поэтому есть упрощения, о которых стоит сказать заказчику:

- Загруженный прайс хранится в `localStorage` браузера — он виден **только на том устройстве, где его загрузили**. В рабочей версии прайс сохраняется на сервере и виден всем посетителям.
- Оптовая заявка никуда не отправляется. В рабочей версии её можно слать на e-mail, в Telegram или в CRM.
- Картинки товаров заменены иконками категорий.

Эти три пункта — то, что нужно будет доделать для боевого запуска (минимальный бэкенд + хранение прайса + приём заявок).

## Структура

```
index.html          витрина магазина
admin.html          загрузка прайса (демо-админка)
assets/css/styles.css
assets/js/data.js    демо-каталог "из коробки"
assets/js/store.js   общий слой данных (каталог, корзина)
assets/js/app.js     логика витрины
assets/js/admin.js   логика загрузки Excel (SheetJS)
Прайс_пример.xlsx    пример файла для загрузки
```

Технологии: чистый HTML/CSS/JS + [SheetJS](https://sheetjs.com) для чтения Excel. Сборка не требуется.
